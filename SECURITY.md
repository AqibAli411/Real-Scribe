# Security Policy

## Supported Versions

Security fixes are provided for the latest `main` branch.

## Reporting a Vulnerability

Please report vulnerabilities privately. Do **not** open a public issue for active security defects.

- Contact: `security@realscribe.local` (replace with your monitored address)
- Include:
  - Impacted endpoint/component
  - Reproduction steps
  - Proof-of-concept (if available)
  - Suggested mitigation

We aim to acknowledge reports within 72 hours and provide mitigation status updates regularly.

## Disclosure Process

1. Triage and verify issue.
2. Develop and validate fix.
3. Prepare release notes and mitigation guidance.
4. Coordinate disclosure window with reporter.
